# Healthcare-Analysis
This project demonstrates how SQL can be used as a powerful tool for healthcare analytics, enabling data-driven decision-making and improving understanding of hospital datasets.


The project uses five different healthcare datasheets, each containing unique information about patients, treatments, hospitals, and medical operations. All datasets were imported into SQL, cleaned, and joined together to build a unified analytical model.

**What This Project Covers**

Importing multiple healthcare datasets into SQL

Cleaning and preparing data for analysis

Joining tables to create a consolidated dataset

Performing deep analysis using SQL functions and logic

Extracting insights related to patient care, hospital performance, and treatment trends

**SQL Techniques Used**

JOIN operations: INNER JOIN, LEFT JOIN, RIGHT JOIN

Aggregate functions: COUNT(), SUM(), AVG(), MIN(), MAX()

CTEs (Common Table Expressions) for simplifying complex queries

Date & String functions for data transformation

Filtering & Grouping: WHERE, GROUP BY, HAVING

Subqueries & Conditional logic for advanced analysis

**Insights Generated**

This analysis helps uncover:

Patient demographics and visit patterns

Treatment effectiveness and frequency

Hospital performance metrics

Billing and cost trends

Category‑wise medical statistics

** Purpose of the Project **
This repository highlights how SQL can be used as a powerful tool for healthcare analytics. It demonstrates real‑world data handling, query optimization, and insight generation — making it useful for learning, portfolio building, and practical healthcare data exploration.
